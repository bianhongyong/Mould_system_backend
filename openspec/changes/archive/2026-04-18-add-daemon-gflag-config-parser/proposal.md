## Why

当前配置解析仅覆盖输入/输出通道，但程序启动与共享内存分配等需要**在启动前**掌握**全部业务模块**的资源调度参数、模块业务参数以及各模块的输入/输出通道。需要改为**仅使用一份总配置文件** `launch_plan.txt`（由**总配置文件路径**定位），由**配置解析器**一次性解析出所有模块的参数；各模块的 `input_channel` 与 `output_channel` **不内联**于 `launch_plan.txt`，而是放在**单独的模块 I/O 配置文件**中，`launch_plan.txt` 内每个模块子字典**包含该文件的路径**。解析器根据总配置路径与各模块 I/O 路径加载并校验全部内容，产出结构化结果，供后续守护进程、共享内存分配等调用方消费。

`launch_plan.txt` 中 `resource`/`module_params` 的**字段名**与工程内 **gflags 已声明的变量名一一对应**（在**某头文件**中通过 `DEFINE_*` 等已注册）；多进程测试程序的**主进程**在验证流程中须根据解析结果对这些 GFLAG **完成赋值**，以贴近真实启动行为。

## What Changes

- 约定**唯一总配置文件** `launch_plan.txt`：文件内以字典形式描述**全部模块**各自的参数块；每块含 `module_name`、`resource`、`module_params`，以及指向独立 I/O 配置文件的**路径字段**（例如 `io_channels_config_path`，名称以最终实现为准）。该 I/O 文件内定义该模块的 `input_channel` 与 `output_channel`。
- 约定**路径语义**：**总配置文件路径**由解析器 API 调用方传入（或项目约定的等价入口）；**各模块 I/O 配置文件路径**写在 `launch_plan.txt` 对应模块子字典中，支持相对路径（相对总配置文件所在目录解析）与绝对路径。
- 实现**统一配置解析器（库/组件）**：根据调用方传入的**总配置文件路径**打开总配置（约定文件名为 `launch_plan.txt` 或实现与文档一致），再按各模块声明路径加载 I/O 文件，解析并校验所有模块；输出按模块划分的结构化结果，并聚合全量通道视图；**不实现守护进程正式程序**及其业务上的子进程拉起、监督等逻辑（测试阶段可单独编写基于 `fork` 的多进程测试程序，见下文）。
- **GFLAG 契约**：`resource` 与 `module_params` 中出现的**每个键名**均须在工程**某头文件**中已通过 gflags **事先声明**；解析器输出保留同名键值，供赋值侧使用。
- **测试**：实现**多进程**测试程序：**主进程**在 `fork` 前（或文档约定的时点）根据解析结果将上述键值写入**同名 GFLAG**；再由主进程 `fork` 子进程做约定验证；该程序**不是**正式守护进程。

## Capabilities

### New Capabilities

- `daemon-module-gflag-bootstrap`：从总配置路径加载 `launch_plan.txt`，再按各模块声明的 I/O 路径加载通道文件，解析全部模块并产出启动/分配所需结构化数据；测试主进程须完成基于解析结果的 GFLAG 赋值验证（解析器交付物；守护进程本体不在此变更内实现）。

### Modified Capabilities

- `channel-topology-config-parser`：通道定义位于各模块**独立 I/O 配置文件**；`launch_plan.txt` 仅通过路径引用；解析器须加载全部引用文件并聚合 `input_channel`/`output_channel`，为共享内存分配提供统一输入数据结构。

## Impact

- 影响代码：`launch_plan.txt` 解析与校验、**模块 I/O 配置文件**加载与校验、路径解析（总配置路径 + 模块路径）、多模块参数与通道聚合逻辑、解析器对外 API；多进程测试程序内 **GFLAG 写入逻辑**（依赖既有头文件中的 gflags 声明）；**不包含**守护进程正式程序、进程拉起、生产环境运行期锁定等实现。
- 影响运行行为：调用方在需要全局视图时调用解析器获取结果；测试主进程演示“解析 → 写 GFLAG → fork”路径。
- 影响运维方式：运维维护**一份**总配置 `launch_plan.txt` 及**每模块一份** I/O 通道配置文件（路径由总配置声明）；配置键须与代码中已注册 GFLAG 同步维护。
- 影响测试：新增基于 `fork` 的多进程测试可执行程序（或测试目标），主进程含 **GFLAG 赋值**步骤。
